# Настройка DNS для GitHub Pages — www.vendettaexternal.com

Этот файл добавлен, чтобы показать точные шаги, которые нужно выполнить у регистратора или в панели DNS (например, Namecheap, GoDaddy, REG.RU, Cloudflare и т.д.), чтобы ваш сайт на GitHub Pages заработал по адресу www.vendettaexternal.com.

Важное: я не могу изменить записи на стороне регистратора или Cloudflare — для этого нужно войти в панель управления вашего домена. Я добавил инструкции в репозиторий, чтобы вы или администратор могли применить их вручную.

1) Что уже есть в репозитории
- В корне репозитория файл CNAME уже содержит одну строку:
  `www.vendettaexternal.com`
  Это правильно: GitHub Pages знает, что ваш кастомный домен — www.vendettaexternal.com.

2) Что нужно добавить в DNS (у регистратора)
- Для поддомена www (обязательно):
  - Тип: CNAME
  - Имя/Host: `www`
  - Значение/Target: `bodyapeee12.github.io`

- Для корневого домена (apex) — чтобы vendettaexternal.com работал/перенаправлял:
  - Тип: A (4 записи)
  - Имя/Host: `@` или пусто
  - Значения (GitHub Pages):
    - `185.199.108.153`
    - `185.199.109.153`
    - `185.199.110.153`
    - `185.199.111.153`
  - Альтернатива: если ваш DNS-провайдер поддерживает ALIAS/ANAME для apex — можно использовать ALIAS на `bodyapeee12.github.io`.

3) Если вы используете Cloudflare
- Убедитесь, что запись `www` имеет режим «DNS only» (серый облачок), а не проксирование (оранжевый облачок). GitHub Pages не работает корректно через включённый Cloudflare proxy для CNAME-записей.

4) В настройках GitHub (от вашего аккаунта)
- В репозитории → Settings → Pages: убедитесь, что в поле Custom domain указано `www.vendettaexternal.com`.
- Включите "Enforce HTTPS" после того, как сертификат будет выпущен GitHub (это может занять до 24 часов).

5) Как проверить, сделали ли вы всё правильно
- Команды (на вашей машине или в онлайн-терминале):
  - Проверить CNAME для www:
    `dig +short CNAME www.vendettaexternal.com`
    Ожидаемый результат: `bodyapeee12.github.io.`

  - Проверить A-записи для корня:
    `dig +short A vendettaexternal.com`
    Ожидаемый результат: четыре IP-адреса GitHub Pages (185.199.108.153 … 185.199.111.153)

  - Через публичный DNS-over-HTTPS (пример):
    `curl -s 'https://dns.google/resolve?name=www.vendettaexternal.com&type=CNAME'`
    `curl -s 'https://dns.google/resolve?name=vendettaexternal.com&type=A'`

- Онлайн-проверки (графический интерфейс):
  - https://dnschecker.org/
  - https://www.whatsmydns.net/

6) Время ожидания
- DNS-пропагация обычно занимает от нескольких минут до нескольких часов, иногда до 24–48 часов.
- Выдача SSL-сертификата GitHub для кастомного домена обычно занимает до 24 часов после корректной DNS-настройки.

7) Частые ошибки и их исправления
- Наличие одновременно A-записи и CNAME для `www` — конфликт: оставьте только CNAME для `www`.
- Если вы видите IP-адреса из Cloudflare (например, 104.16.x.x, 172.67.x.x) для `www` — значит включено проксирование Cloudflare; переключите на DNS-only.
- Провайдер может не позволять CNAME на поддомен у apex — для корня используйте A-записи или ALIAS/ANAME.
